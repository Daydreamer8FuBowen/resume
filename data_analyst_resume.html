数据治理开发工程师

北京联通｜北京市感知管理服务平台
2025.06 – 2025.12

项目背景：平台需要对 MySQL、Kafka、文件上传等多数据源数据进行统一治理并存入 TDengine ，为数据分析与业务查询提供支撑。

核心贡献：

  • Kafka 消息分区优化：按数据模型对 Kafka 消息进行分区与路由设计，提升消费并行度并缓解热点分区问题，在高峰场景下数据链路吞吐能力提升 约 2–6 倍 。
  • TDengine 批量写入优化：根据业务实时性需求较低的特点，将写入方式由逐条写入优化为批处理写入，并调优批大小与 flush 策略，使数据库写入吞吐提升 约 3–5 倍 。
  • Canal 增量同步优化：引入 Canal 监听 MySQL binlog 并直连 Kafka，将数据同步方式由轮询拉取改为增量变更驱动，减少 MySQL 查询压力，读负载降低 约 30–50% 。
  • 数据治理与查询优化：使用 Python 完成数据清洗与质量校验（缺失值、异常值、重复数据等），并通过 SQL 与索引优化提升数据查询效率。

技术栈：

  • SpringBoot
  • Kafka
  • TDengine
  • Canal
  • SQL
  • Python
